**Твой долгожданный отпуск уже на носу!**

Последний пункт в списке дел — купить новые солнцезащитные очки. Ты заходишь в небольшой, но очень стильный магазинчик, находишь идеальную пару и разговорился с владельцем.

И тут ты с удивлением узнаешь, что у этого крутого места до сих пор нет своего интернет-магазина или мобильного приложения. Весь бизнес держится на случайных прохожих.

Владелец в восторге и соглашается. Дизайн уже есть. Теперь твоя задача — сдержать обещание и запустить этот MVP, реализовав самый важный функционал: от главной страницы до оформления заказа.

### Техническое задание

**Цель:** создать мобильное приложение интернет-магазина с каталогом товаров, корзиной, формой заказа и интеграцией с API.

## **0. Дизайн-макет**

https://www.figma.com/design/47TArw335ATcFSWuHar9KN/Module-3-React-Native-%7C-gluestack-ui-v2.0-Design-Kit--Share-?node-id=13607-22813&t=E7hyrT2Z9K0XSyuE-1

## **1. Главная страница (`HomeScreen`)**

- Отображение **списка товаров** (загрузка через API)
- Компоненты:
  - `ProductCard` — карточка товара
  - `ProductList` — список товаров
- Использование `FlatList` для прокрутки
- Skeleton Loader для данных (например, через `react-native-skeleton-placeholder`)

**Темы:**

- Основные компоненты
- API и загрузка данных

## **2. Страница продукта (`ProductScreen`)**

- Динамическая навигация через **React Navigation**
- Загрузка данных о продукте по `id`
- Отображение **деталей товара**, цены, отзывов
- Кнопка "Добавить в корзину" (`Add to bag`), которая меняет свое состояние на "Добавлено" (`Added`) после нажатия.

**Темы:**

- Навигация 
- API-запросы
- Управление состоянием

## **3. Корзина (`CartScreen`)**

- Хранение данных о товарах в **Redux**
- Отображение товаров в корзине через `FlatList`
- Возможность изменять количество (`+`/`-`) и удалять товар.
- Общая стоимость, количество товаров
- Форма оформления заказа (`TextInput` + `Button`)

**Темы:**

- Redux
- AsyncStorage для сохранения данных


## **4. Оформление заказа (`CheckoutScreen`)**

- Выбор способа оплаты из списка.
- Кнопка "Оплатить" (`Pay`) для завершения заказа.
  
*   **`OrderAccepted` (Заказ принят):**
-   Экран подтверждения с сообщением об успешном оформлении заказа.


**Темы:**

- Redux
- AsyncStorage для сохранения данных
